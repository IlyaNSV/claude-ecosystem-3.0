# READ / P5-adversarial — Адверсариальный ввод как зона безопасности веб-модуля

> **Прогон 2, приоритет P5.** Глубокое чтение первоисточников по indirect prompt injection (IPI).
> Дата: 2026-07-12. Все числа — с точной ссылкой и условиями замера.
> **Взвешивание — по decision-delta × (1 − indirectness), НЕ по тиру S-лестницы** (per бриф).

---

## 0. Главный вывод за 30 секунд

Мы проектируем модуль, который: **(1)** ходит в недоверенную сеть, **(2)** читает страницы, **(3)** **грейдит их LLM-судьёй**, **(4)** результат кладёт в контекст-файлы (CLAUDE.md / память), которые **авто-загружаются как инструкции каждую сессию**.

Литература говорит три вещи, и все три бьют по разным звеньям этой цепочки:

1. **Атака на агента, читающего внешний контент, работает** — на фронтир-моделях, в реалистичных задачах, без всякой магии. GPT-4o: targeted ASR **47.7–57.7 %** (AgentDojo, NeurIPS 2024). Claude 3.5 Sonnet: **33.9 %**.
2. **Судья — это отдельная, недооценённая мишень, и его НЕ защищает ни одна из «работающих» защит.** Все сильные защиты (tool filter, CaMeL, plan-then-execute) охраняют **поток управления и вызовы инструментов**. Судья инструментов не вызывает — он выносит **суждение**. Суждение хакается: «нулевая модель» получает **86.5 %** LC win-rate у GPT-4-судьи, и **76.8 % вообще без оптимизации** (ICLR 2025). Атака на суждение — это наш ОСНОВНОЙ риск, а не эксфильтрация.
3. **Профилактика не масштабируется — только сдерживание.** Anthropic на своём же стеке: 23.6 % → 11.2 % с митигациями, и ~1 % под адаптивным атакующим; их формулировка — «**No browser agent is immune to prompt injection**». Защиты, показавшие 0 % на статике, дают **28–64 %** под чёрноящичной адаптивной атакой (AutoDojo, препринт 06.2026).

**Что это меняет в решении:** дизайн веб-модуля должен исходить не из «отфильтруем инъекции», а из «**инъекция пройдёт; сделаем так, чтобы её проход ничего не решал**». Конкретика — §7.

## 1. Терминологическая развилка, которая и есть суть (Greshake et al.)

**Источник:** Greshake, Abdelnabi, Mishra, Endres, Holz, Fritz. *«Not what you've signed up for: Compromising Real-World LLM-Integrated Applications with Indirect Prompt Injection»*, arXiv **2302.12173**, **AISec@ACM CCS 2023** (peer-reviewed workshop). Первоисточник термина.

Дословно из абстракта:

> «So far, it was assumed that the user is directly prompting the LLM. But, what if it is not the user prompting? We argue that LLM-Integrated Applications blur the line between data and instructions. We reveal new attack vectors, using **Indirect Prompt Injection**, that enable adversaries to **remotely (without a direct interface)** exploit LLM-integrated applications by strategically injecting prompts into data likely to be retrieved.»

**Ключевое различение (и почему бриф прав):**

| | Jailbreak / direct PI | **Indirect PI** |
|---|---|---|
| Кто вводит текст | Сам пользователь | **Третья сторона**, заранее, в данные |
| Против кого | Против политик провайдера | **Против пользователя и его агента** |
| Что ломается | Отказ модели генерировать X | **Следование легитимной инструкции** |
| Наш аналог | «уговори Claude сказать плохое» | «**Claude прочитал страницу и перестал делать то, что я просил**» |

Это не одна и та же угроза, и защиты от них разные. Наш модуль **не подвержен** jailbreak-риску в интересном смысле (нам не нужно, чтобы модель отказывалась) — и **полностью подвержен** IPI: он по построению читает недоверенный текст третьих лиц.

**Таксономия доставки** (их же): *passive* (инъекция лежит на веб-странице/в репозитории и ждёт, когда её достанут ретривом — **это ровно наш случай**), *active* (письмо), *user-driven* (соцынж копипастой), *hidden* (многоступенчатые/кодированные/в других модальностях).

**Категории ущерба:** information gathering (эксфильтрация), fraud, intrusion, **malware / «черви» (self-replicating prompts)**, **manipulated content** (предвзятые сводки, дезинформация, подавление источников, скрытая реклама), availability.

> **Категория «manipulated content» — это про нас.** Не RCE, не утечка ключей. Порча **выхода исследования**: страница уговаривает агента считать её авторитетной, а конкурента — нет.

**Демонстрации на живых системах (2023):** Bing Chat (GPT-4) — эксфильтрация через URL-параметры, фишинг, подсовывание вредоносных markdown-ссылок, **подавление источников** (заставили не цитировать NYT), искажение фактов, DoS-порча поисковых запросов; GitHub Copilot — инъекция в комментарий пакета отравляет автодополнение и **живёт в контекстном окне, пока его не вычистят**.

**Их вывод про защиты, дословно:**

> «the defensive approach seems to follow a **'Whack-A-Mole'** style»
> «it is currently hard to imagine a foolproof solution»
> «**effective mitigations of these emerging threats are currently lacking**»

Отдельно — «detection dilemma»: фильтровать ретривнутый текст слабой моделью → пропустим сложные/кодированные атаки; фильтровать сильной моделью → **воспроизвели исходную уязвимость внутри фильтра**. (Это ровно та ловушка, в которую можно въехать, поставив «LLM-детектор инъекций» перед нашим LLM-судьёй.)

- **Tier:** S1-S2 (peer-reviewed workshop AISec@CCS; де-факто канон, >1000 цитирований).
- **Indirectness → наш вопрос:** **small**. Bing Chat = LLM, читающий веб-страницы. Copilot = кодовый агент. Оба — наш класс. Числа отсутствуют (это qualitative-работа) → числа берём из §2-§3.

## 2. AgentDojo — сколько именно ломается (числа)

**Источник:** Debenedetti, Zhang, Balunović, Beurer-Kellner, Fischer, Tramèr. *«AgentDojo: A Dynamic Environment to Evaluate Prompt Injection Attacks and Defenses for LLM Agents»*, arXiv **2406.13352v3**, **NeurIPS 2024 Datasets & Benchmarks**.

**Что меряли:** 97 реалистичных user-задач + **629 security-тест-кейсов**, 4 набора (workspace / Slack / travel / banking). Метрики: *benign utility* (решает ли задачу без атаки), *utility under attack*, ***targeted ASR*** (выполнил ли агент **задачу атакующего**). Атака-чемпион — «**Important instructions / Important message**»: в тул-выдачу вставляется сообщение, что перед исходной задачей надо выполнить вот эту.

### Таблица 3 (сверена двумя независимыми вычитками HTML-версии; 95 % ДИ в скобках)

| Модель | Benign utility | Utility under attack | **Targeted ASR** |
|---|---|---|---|
| **GPT-4o** | **69.00 %** (±3.61) | 50.08 % (±3.91) | **47.69 %** (±3.90) |
| **Claude 3.5 Sonnet** | **78.22 %** (±3.23) | 51.19 % (±3.91) | **33.86 %** (±3.70) |
| GPT-4 Turbo | 63.43 % (±3.76) | 54.05 % (±3.89) | 28.62 % (±3.53) |
| Claude 3 Sonnet | 53.10 % (±3.90) | 33.23 % (±3.68) | 26.71 % (±3.46) |
| Gemini 1.5 Pro | 45.63 % (±3.89) | 28.93 % (±3.54) | 25.60 % (±3.41) |
| Llama 3 70B | 34.50 % (±3.71) | 18.28 % (±3.02) | 20.03 % (±3.13) |
| Gemini 1.5 Flash | 36.09 % (±3.75) | 34.18 % (±3.71) | 12.24 % (±2.56) |
| **Claude 3 Opus** | 66.61 % (±3.69) | 52.46 % (±3.90) | **11.29 %** (±2.47) |
| GPT-3.5 Turbo | 33.86 % (±3.70) | 34.66 % (±3.72) | 8.43 % (±2.17) |
| **Command-R+** | **25.44 %** (±3.40) | 25.12 % (±3.39) | **0.95 %** (±0.76) |

### Три вывода, которые реально меняют решение

**(а) Inverse scaling: способнее ⇒ легче взломать.** Самая полезная модель (GPT-4o, 69 %) — и самая пробиваемая (47.7 %). Самая «стойкая» (Command-R+, 0.95 %) — потому что она **вообще ничего не делает** (benign utility 25 %). Цитата авторов: *«more capable models tend to be easier to attack»*. То же самое отмечал Greshake («inverse scaling»).

> **contradicts_our_prior = TRUE**, если приор был «мы на Opus 4.8, мы умные, нас не проведёт мусорная страница». Способность следовать инструкциям **и есть** атакуемая поверхность: агент, который хорошо слушается, хорошо слушается **любого**.
> **Честная оговорка:** это закон *внутри среза 2024 г.*, не закон во времени. Claude 3 Opus (11.3 %) стойче Claude 3.5 Sonnet (33.9 %) при сопоставимой полезности → провайдер может двигать эту точку. Публичных чисел AgentDojo для Claude 4.x / Opus 4.8 я **не нашёл** (§9).

**(б) Атака НЕ обязана всё ломать, чтобы быть фатальной для нашего кейса.** «Всего 33.9 %» у Claude 3.5 Sonnet звучит терпимо — но это вероятность **на одну адверсариальную страницу**. Модуль добора читает десятки страниц за прогон. При независимости, 5 адверсариальных страниц из 50 при p=0.34 дают ~**87 %** вероятности хотя бы одного успешного захвата за прогон. Условие «страница адверсариальна» — не измерено ни в одной работе (§9, главный пробел).

**(в) 17 % кейсов не изолируемы в принципе.** Дословно: инструменты, нужные для решения задачи, **достаточны и для атаки** — *«this is true for 17% of our test cases»*. Т.е. даже идеальная изоляция инструментов оставляет дыру там, где «читать страницы» = и есть возможность атакующего.

- **Tier:** S1 (NeurIPS D&B, с кодом и открытым бенчмарком).
- **Indirectness:** **small**. Не CLAUDE.md-файлы, не Claude Code. Но: реальный многошаговый агент, реальные инструменты, недоверенные данные в тул-выдаче — то же самое, что WebFetch → контекст → действие. Перенос почти прямой.

## 3. AgentDojo — защиты: что РАБОТАЕТ, что заявлено, что дорого

### Таблица 5 (GPT-4o, атака «Important instructions»; сверена дважды)

| Защита | Benign utility | Utility под атакой | **Targeted ASR** | Цена полезности |
|---|---|---|---|---|
| **No defense** | 69.0 % (±3.6) | 50.01 % (±3.9) | **57.69 %** (±3.9) | — |
| **Tool filter** (изоляция инструментов) | **73.13 %** (±3.5) | 56.28 % (±3.9) | **6.84 %** (±2.0) | **+4.1 п.п. (бесплатно!)** |
| **Repeat prompt** (sandwiching) | **85.53 %** (±2.8) | **67.25 %** (±3.7) | 27.82 % (±3.5) | **+16.5 п.п. (!)** |
| **Delimiting** (spotlighting/разделители) | 72.66 % (±3.5) | 55.64 % (±3.9) | **41.65 %** (±3.9) | +3.7 п.п. |
| **PI detector** (BERT-классификатор) | **41.49 %** (±3.9) | 21.14 % (±3.2) | 7.95 % (±2.1) | **−27.5 п.п. (−40 % отн.)** |

**Определения (дословно из §4.3):**
- *Tool filter*: «the LLM first restricts itself to a set of tools required to solve a given task, **before observing any untrusted data**».
- *Repeat prompt*: «repeats the user instructions after each function call».
- *Delimiters*: «format all tool outputs with special delimiters, and prompt the model to ignore instructions within these».
- *PI detector*: «a BERT classifier trained to detect prompt injection on each tool call output, and **aborts the agent** if anything has been detected».

### Читаем эту таблицу как инженеры

1. **Победитель — архитектурная изоляция, а не промптинг.** Tool filter: ASR 57.7 → **6.8 %**, и полезность при этом **выросла**. Это лучший обмен в таблице с огромным отрывом. Механизм: фиксируем набор инструментов **до** чтения недоверенных данных → инъекция физически не может дать агенту новую способность.
2. **Промпт-защиты — половинчатые.** Разделители/spotlighting: 57.7 → 41.7 %. Это **−28 % относительно**, при том что маркетинг spotlighting-подхода (Microsoft, Hines et al., arXiv 2403.14720) обещал падение ASR «ниже 2 %». На реалистичном агентном бенче — 41.7 %. **Классический пример «заявлено ≫ работает»**: защита, померенная на своей же узкой постановке, разваливается на чужой.
3. **Детектор — самая дорогая и самая обманчивая покупка.** ASR падает красиво (57.7 → 7.95 %), но **бенайн-полезность рушится 69 → 41.5 %** (−40 % относительно), потому что детектор **обрывает агента** на ложных срабатываниях. Плюс — см. §5: против адаптивной атаки детекторы сдуваются.
4. **Repeat prompt — дешёвый бонус, но самообманчивый.** Он **повышает** полезность (85.5 %!) и вдвое режет ASR. Авторы сами предупреждают, дословно: *«Repeating the user prompt after a tool call is a reasonable defense for our attack, but it is **unlikely to withstand adaptive attacks** (e.g., an injection that instructs the model to ignore future instructions)»*. Т.е. брать — да; полагаться — нет.

> **contradicts_our_prior = TRUE** для рефлекса «поставим детектор промпт-инъекций на входе веб-модуля и закроем тему». Детектор — **худший** обмен полезность/безопасность из измеренных, и он же первым падает под адаптивной атакой.

### Что «работает по построению» (за пределами AgentDojo-таблицы)

- **CaMeL** — Debenedetti et al., *«Defeating Prompt Injections by Design»*, arXiv **2503.18813** (Google DeepMind + ETH; код открыт). Дословно: *«CaMeL explicitly extracts the control and data flows from the (trusted) query; therefore, the untrusted data retrieved by the LLM **can never impact the program flow**… uses a notion of a **capability** to prevent the exfiltration of private data… We demonstrate effectiveness of CaMeL by solving **77 % of tasks with provable security** (compared to **84 %** with an undefended system) in AgentDojo.»*
  → **Цена доказуемой безопасности ≈ 7 п.п. полезности.** Это лучший из известных обменов. Механизм — не «уговорить модель», а **вынести поток управления из-под влияния данных**.
- **Design Patterns** — Beurer-Kellner et al., arXiv **2506.08837**. Ядро, дословно:
  > «**Once an LLM agent has ingested untrusted input, it must be constrained so that it is impossible for that input to trigger any consequential actions.**»

  Шесть паттернов: *Action-Selector* (агент только выбирает из предопределённых вызовов), *Plan-Then-Execute* (план фиксируется **до** чтения данных → «tool outputs cannot inject instructions that make the agent deviate from its plan»), *LLM Map-Reduce* (изолированный агент на каждый кусок недоверенных данных), *Dual LLM* (привилегированный планировщик + **карантинный LLM без инструментов**), *Code-Then-Execute*, *Context-Minimization* (вычищать лишнее из контекста между ходами).
  Их же приговор общему случаю: *«it is unlikely that general-purpose agents can provide meaningful and reliable safety guarantees»*, а эвристические улучшения «are likely to remain **heuristic in nature — and thus inherently brittle**».

- **Tier:** S1 (AgentDojo/NeurIPS), S2 (CaMeL, Design Patterns — препринты с кодом, но от авторов самого бенчмарка).
- **Indirectness:** **small**.

## 4. InjecAgent — тривиальный «hacking prompt» удваивает ASR

**Источник:** Zhan, Liang, Ying, Kang. *«InjecAgent: Benchmarking Indirect Prompt Injections in Tool-Integrated Large Language Model Agents»*, arXiv **2403.02691**, **Findings of ACL 2024**.

**Состав:** **1 054 тест-кейса**, **17** user-инструментов, **62** атакующих инструмента, **30** LLM-агентов. Два класса намерения: *direct harm to users* и *exfiltration of private data*. Два режима: **base** (только вредоносная инструкция в тул-выдаче) и **enhanced** (+ «hacking prompt» вида «игнорируй предыдущие указания»).

**Числа из абстракта (авторитетно, дословно):**

> «We evaluate 30 different LLM agents and show that agents are vulnerable to IPI attacks, with **ReAct-prompted GPT-4 vulnerable to attacks 24 % of the time**. Further investigation into an enhanced setting, where the attacker instructions are reinforced with a hacking prompt, shows additional increases in success rates, **nearly doubling the attack success rate on the ReAct-prompted GPT-4**.»

→ **24 % → ~47 %** от одной дописанной строчки. Стоимость атаки для противника — ноль; прирост — двукратный.

**Второй результат (важен для дизайна):** файн-тюнинг на агентные траектории резко снижает уязвимость — «fine-tuned GPT-4 and GPT-3.5 demonstrate **greater resilience** to these attacks, with significantly lower ASRs» (порядка единиц процентов). Т.е. **уязвимость — свойство промпт-режима (ReAct), а не «природы LLM»**: как модель обучена принимать тул-выдачу, так она её и трактует.

> **Что это значит нам:** мы не контролируем файн-тюнинг Claude. Значит, наш единственный рычаг — **как мы подаём недоверенный текст в контекст**. Anthropic это же и советует явно: третьесторонний контент доставлять **внутри `tool_result`-блоков, никогда — в system-prompt или обычный user-текст**; Claude обучен относиться к инструкциям внутри tool-результатов со скепсисом. Наш модуль ОБЯЗАН уважать эту границу: **никакого `curl page > context`, никакого «вставь текст страницы в промпт судьи как будто это часть задания»**.

- **Tier:** S1 (Findings ACL 2024, открытый бенчмарк).
- **Indirectness:** **small** (агент + тул-выдача + недоверенный контент = наш WebFetch-контур).
- **Честно:** таблицу по-модельно (Claude-2, Llama-2, Vicuna) **достоверно вытащить не удалось** — многоуровневая шапка ломается при конверсии в markdown, ячейки поехали. Числа по-модельно **не цитирую**. Цитирую только абстракт + вывод про файн-тюнинг (подтвердился в двух независимых вычитках).

## 5. Адаптивные атаки: статичный бенчмарк ЗАВЫШАЕТ защиту

Это тот класс результатов, который обесценивает любое наше будущее «мы прогнали тест — у нас 0 %».

**(а) Zhan et al., *«Adaptive Attacks Break Defenses Against Indirect Prompt Injection Attacks on LLM Agents»*, arXiv 2503.00061 (NAACL Findings 2025).**
Восемь защит: детектор на DeBERTaV3, LLM-детектор, perplexity-фильтр, instructional prevention, data-prompt isolation, sandwich prevention, paraphrasing, adversarial fine-tuning.
Дословный вывод: *«Adaptive attacks consistently achieve success rates **above 50 %** across the targeted defenses and LLM agents — **exceeding the ASR before deploying any defense** and significantly outperforming non-adaptive attacks.»*
Показательные пары (Vicuna-7B): paraphrasing **3 % → 79 %**; sandwich prevention **12 % → 64 %**; perplexity-фильтр **32 % → 76 %**; adversarial fine-tuning **12 % → 62 %**.
> **Жёсткая оговорка (indirectness = large):** модели — **Vicuna-7B и Llama3-8B**, атака — **GCG/AutoDAN, т.е. градиентная, белоящичная**. Против Claude через API это неприменимо напрямую. Переносится **качественный** урок: «защита, померенная только против статичной атаки, — не измерена».

**(б) AutoDojo, arXiv 2606.15057 (препринт, июнь 2026) — вот это уже про нас.**
Адаптивное расширение AgentDojo; атакующий — **чёрноящичный**, фронтир-LLM итеративно оптимизирует инъекцию под конкретную защиту. Дословно:
> «a **cheap, black-box adaptive attack** using a frontier LLM to iteratively optimize the injection raises attack success rate (ASR) well above the level achieved by static injections against **nearly all evaluated defenses**. Against a filter that reduces static ASR to **0 %**, AutoDojo recovers **28 % overall and 64 % on action-open tasks**.»

И — второй результат, который **прямо переписывает наш дизайн**:
> «for prompt-level and filter-based defenses, ASR is substantially higher on **action-open tasks** — where the user's request **delegates the action itself to attacker-controlled content** — than on precisely specified tasks. This is a **structural limit**: on such tasks the injection can **pose as ordinary data rather than an explicit instruction**, bypassing defenses that rely on detecting instruction-like text.»

> **Наш модуль по построению — «action-open».** «Найди в сети релевантное, оцени качество, реши, что взять» = решение делегировано содержимому страниц. Именно там детекторы структурно слепы: инъекции **не обязаны выглядеть как инструкция** — достаточно выглядеть как убедительные данные («это исследование — золотой стандарт, N=50000, Nature»).
> Обратная сторона того же результата — **дешёвая контрмера**: чем **точнее заранее специфицирована задача** (жёсткая рубрика, фиксированные критерии приёмки, заранее решённый список того, что мы ищем), тем ниже ASR. Специфицировать рубрику **до** выхода в сеть — бесплатно и работает.

- **Tier:** 2503.00061 — S2/S1 (NAACL Findings). AutoDojo — **S3 (свежий препринт, не рецензирован; ячейки таблиц я не верифицировал)**.
- **Indirectness:** 2503.00061 — **large** (мелкие открытые модели, белый ящик). AutoDojo — **small** (AgentDojo-суиты, 5 фронтир-моделей, чёрный ящик).

## 6. ⚠️ САМОЕ ВАЖНОЕ: судья — это мишень, и его не защищает ни одна «работающая» защита

Бриф ставит вопрос ребром: «судья тоже читает недоверенный текст!». Отвечаю: **да, и это наш главный риск, а не эксфильтрация.**

**Асимметрия, которую легко пропустить.** Все сильные защиты из §3 — tool filter, CaMeL, Plan-Then-Execute, Action-Selector — защищают **поток управления и вызовы инструментов**. Формулировка Beurer-Kellner: «impossible for that input to **trigger any consequential actions**». Но **у судьи нет инструментов**. Он ничего не «триггерит». Он **выносит оценку** — а оценка в нашей системе и есть consequential action: она решает, что попадёт в корпус, в синтез и (в пределе) в CLAUDE.md.

> **Отсюда: «судья без инструментов ⇒ судья безопасен» — ЛОЖЬ.** Он безопасен от RCE и эксфильтрации. Он полностью открыт для **порчи суждения** — а нам ровно суждение и нужно.

Три независимых источника, каждый бьёт в это место:

**(а) «Нулевая модель» ломает GPT-4-судью — БЕЗ оптимизации.**
*«Cheating Automatic LLM Benchmarks: Null Models Achieve High Win Rates»*, arXiv **2410.07137**, **ICLR 2025**.
Модель, которая **всегда выдаёт одну и ту же константную строку, не относящуюся к вопросу**, получает:
- **86.5 %** LC win-rate на **AlpacaEval 2.0** (аннотатор — GPT-4);
- **83.0** на Arena-Hard-Auto;
- **9.55** на MT-Bench.

И критично для нас: *«The structured cheating response is highly effective in confusing GPT-4 based auto-annotators, achieving a substantial **76.8 % LC win rate on AlpacaEval 2.0 even without optimization**, exploiting syntactic weaknesses in LLMs by **counterfeiting instructions and outputs**»*. Оптимизация (random search) добивает до 86.5 %.
→ **76.8 % — это рукописная строка. Чёрный ящик. Переносимая. Против фронтир-судьи.** Ровно такую можно положить в `<div style="display:none">` на странице.

**(б) JudgeDeceiver — оптимизированная инъекция в кандидата.**
Shi et al., *«Optimization-based Prompt Injection Attack to LLM-as-a-Judge»*, arXiv **2403.17710**, **ACM CCS 2024** (код открыт).
Инъекция вставляется в **контролируемый атакующим кандидатный ответ**, и судья выбирает его **независимо от того, какие есть другие кандидаты**. Средние ASR: **89.2 % / 88 %** (Openchat-3.5), **90.8 % / 93.2 %** (Mistral-7B), **98.9 % / 98.1 %** (Llama-2-7B), **97.6 % / 97 %** (Llama-3-8B, на MT-Bench / LLMBar).
> **Честно (indirectness = large):** это **открытые 7-8B судьи** и **оптимизационная** (градиентная/суррогатная) атака. Пробитие фронтир-судьи (GPT-4/Claude) в этой работе **не показано**. Ценность для нас — не число, а **механизм**: «позиционная состоятельность» атаки (PAC 79-98 %) означает, что подмена суждения устойчива к перестановке кандидатов, т.е. это не шум ранжирования, а захват.

**(в) Preference Manipulation Attacks — на ПРОДАКШН-системах, включая Claude.**
Nestaas, Debenedetti, Tramèr, *«Adversarial Search Engine Optimization for Large Language Models»*, arXiv **2406.18382** (та же лаборатория, что AgentDojo).
Дословно: *«carefully crafted **website content** or plugin documentations can trick an LLM to **promote the attacker products and discredit competitors**… We demonstrate our attacks on **production LLM search engines (Bing and Perplexity)** and plugin APIs (for **GPT-4 and Claude**).»*
Числа: в Bing Copilot атакованная (вымышленная!) камера рекомендуется в **2.5×** чаще бенайн-конкурента и выходит на **59.4 %** рекомендаций против **57.9 %** у реальных Nikon/Fujifilm — т.е. **фейк с инъекцией догоняет настоящие бренды**. В выборе плагинов: до **7.2×** роста вероятности выбора у GPT-4 Turbo; у **Claude 3 Opus** — от 0 % до **>90 %** в зависимости от конфигурации. Доставка — **скрытый текст (белый шрифт, размер 1 px)** прямо на странице.
И «дилемма заключённого»: когда атакуют все, **проигрывают все** — качество выдачи LLM деградирует глобально.

> **Это наш модуль в чистом виде.** «LLM выбирает из конкурирующего третьесто­роннего контента» — буквальное определение того, что делает наш судья. И у авторов страниц есть **прямой экономический стимул** это делать (GEO-индустрия уже существует). Т.е. злонамеренная страница — **не экзотика из threat-model, а бизнес-модель**.

**Итог по §6 (RULE):** судья, читающий недоверенную страницу, — атакуемое звено; атака на него **не требует ни одного вызова инструмента**, поэтому все инструмент-центричные защиты (tool filter, CaMeL, sandbox, permissions) **дают нам ровно ноль защиты здесь**. Три независимых источника (ICLR 2025 + CCS 2024 + продакшн-демо на Claude/Bing), два из них на фронтир-судьях в чёрном ящике.

## 7. Что делать веб-модулю: перевод в инженерные правила

### RULE-1. Веб-текст — данные, никогда не инструкции. Ни разу, нигде.
Недоверенный контент подаётся **только** как tool-результат (или его эквивалент), **никогда** — в system-prompt, в CLAUDE.md, в текст задания судье как часть промпта. Никаких `curl url | claude`. Никакого «вот содержимое страницы, следуй ему».
*Опора:* Greshake (AISec@CCS'23) + Anthropic guidance (tool_result-канал) + InjecAgent (уязвимость — свойство режима подачи). ≥2 независимых. **RULE.**

### RULE-2. Инструменты фиксируются ДО первого сетевого чтения.
Tool filter — единственная защита с ASR 57.7 → **6.8 %** и **положительной** ценой полезности (+4.1 п.п.). Веб-модуль объявляет свой набор инструментов заранее и **не приобретает новых после того, как прочитал хоть байт из сети**. Ни при каких «страница просит запустить…».
*Опора:* AgentDojo Table 5 (S1, код, бенчмарк). **RULE.**

### RULE-3. Рубрика оценки фиксируется ДО выхода в сеть (борьба с «action-open»).
Не «прочитай и реши, что важно», а «прочитай и заполни **предопределённый** формуляр: соответствует ли критерию A/B/C, да/нет, цитата-подтверждение». Это переводит задачу из *action-open* (где инъекция маскируется под данные и обходит фильтры) в *precisely specified* (где ASR структурно ниже).
*Опора:* AutoDojo (S3-препринт, 06.2026) + Plan-Then-Execute (Design Patterns, S2). **HYPOTHESIS→RULE** (дёшево, риск нулевой, ожидаемая польза высокая; но эффект у нас не измерен → см. TEST-1).

### RULE-4. Выход судьи — минимальная схема, а не свободный текст.
Судья возвращает **enum/число/фиксированные теги**, а не абзац прозы. Причина: **выход судьи сам является данными под влиянием атакующего**. Если оркестратор потом читает свободный «rationale» судьи — инъекция просто **проехала через карантин** в привилегированный контекст (пропагация через посредника). Карантинный LLM в Dual-LLM-паттерне возвращает **символьную переменную**, а не текст, которому доверяют.
*Опора:* Design Patterns (Dual LLM: «This LLM cannot use any tools»), Greshake (persistence в контексте). **RULE** для принципа; **HYPOTHESIS** для «схема снижает ASR численно» → TEST-2.

### RULE-5. Ни один байт веб-происхождения не попадает в авто-загружаемый канал автоматически.
**Это самое опасное место всего дизайна.** CLAUDE.md и MEMORY.md **авто-инжектятся в каждую сессию как инструкции наивысшего доверия**. Модуль «умного добора из сети» по замыслу пишет туда находки. Значит цепочка «недоверенная страница → судья → синтез → CLAUDE.md» — это **машина по отмыванию недоверенного текста в привилегированный инструкционный канал, с персистентностью**. Одна успешная инъекция становится **постоянной** и переживает сессии (ровно «worming» / «persists in context until purged» у Greshake; ровно механика EchoLeak — контент письма попал в контекст и вытащил данные).
→ **Между веб-выходом и любым авто-загружаемым файлом обязателен человеческий/main-модельный гейт.** Никакого авто-коммита находок в CLAUDE.md/память. (Это, кстати, ровно та граница, что у нас уже стоит в Autoflow: «merge в main — всегда владелец». Нужно распространить её на «запись в контекст-файлы».)
*Опора:* Greshake (persistence, worming) + EchoLeak CVE-2025-32711 (реальный zero-click, CVSS 9.3) + Anthropic guidance (untrusted → только tool_result). **RULE.**

### RULE-6. Проектируем на сдерживание, не на предотвращение. Остаточный риск ≠ 0 — по признанию вендора.
Anthropic, Claude for Chrome (на **123** адверсариальных тест-кейсах, **29** сценариев): *«Browser use without our safety mitigations showed a **23.6 % attack success rate**»* → *«we reduced the attack success rate of 23.6 % to **11.2 %**»*; на «челлендж»-наборе из 4 браузер-специфичных атак — *«from **35.7 % to 0 %**»*. В более поздней публикации против адаптивного Best-of-N-атакующего (**100 попыток на среду**) остаётся *«A **1 %** attack success rate — while a significant improvement — still represents meaningful risk»*, и прямым текстом: *«**No browser agent is immune to prompt injection**»*.
Claude Code docs подтверждают наличие карантина — *«**Isolated context windows**: Web fetch uses a separate context window to avoid injecting potentially malicious prompts»* — и тут же: *«While these protections significantly reduce risk, **no system is completely immune to all attacks**»*.
→ Значит: **радиус поражения** должен быть мал по построению. Веб-модуль работает без записи в репо, без сети наружу (кроме чтения), без секретов в контексте, результат — в песочницу для ревью.
*Опора:* Anthropic (вендор, indirectness = **none** — это буквально Claude, читающий веб) + Claude Code docs. **RULE.** *Оговорка:* вендорские цифры self-serving; но именно поэтому ценно, что даже они не заявляют 0 %.

### Чего делать НЕ надо (антипаттерны, доказательно)
- ❌ **«Поставим детектор инъекций перед судьёй.»** Худший обмен: −27.5 п.п. полезности (69 → 41.5 %), и он же первым рушится под адаптивной атакой (0 % статики → 28-64 % у AutoDojo; 3 % → 79 % у paraphrasing в 2503.00061). Плюс «detection dilemma» Greshake: сильный детектор = та же уязвимость внутри детектора.
- ❌ **«Обернём страницу в разделители и попросим не слушаться.»** 57.7 → 41.7 %. Половина атак проходит. Обещали <2 % — не воспроизводится на агентном бенче.
- ❌ **«Мы на топ-модели, нас не проведёшь.»** Inverse scaling: топ-модель = топ-ASR (GPT-4o 69 % полезности / 47.7 % ASR).
- ❌ **«У судьи нет инструментов, значит он безопасен.»** См. §6 — атака на суждение инструментов не требует.

## 9. Честно о ненайденном и о слабых местах (honest limits)

1. **Внутреннее противоречие в AgentDojo, которое авторы не объясняют.** GPT-4o «no defense» targeted ASR = **47.69 %** в Table 3 и **57.69 %** в Table 5 (обе — атака «Important instructions», обе сверены двумя независимыми вычитками HTML v3). В тексте нет фразы, примиряющей эти цифры. Я привожу обе и оперирую диапазоном **47.7-57.7 %**. Не подгоняю.
2. **Таблицу InjecAgent по-модельно достоверно вытащить не удалось** (многоуровневая шапка разъезжается при конверсии в markdown; полученные ячейки были явно битые — напр. «GPT-4 base total 97.7 %» при абстракте «24 %»). Поэтому **числа по Claude-2 / Llama-2 / Vicuna НЕ привожу**. Только абстракт + вывод про файн-тюнинг.
3. **Нет публичных чисел AgentDojo для Claude 4.x / Opus 4.8 — т.е. для НАШЕЙ модели.** Страница результатов бенчмарка (`agentdojo.readthedocs.io/.../results/`, `agentdojo.spylab.ai/results`) отдала 404. Цифра «Claude 3.7 Sonnet — 7.3 % ASR» встретилась только во **вторичном** пересказе, к первичной таблице я её не привязал → **как находку не использую**. Практический смысл: наши ASR-числа — это оценка **порядка величины**, а не наш стек.
4. **JudgeDeceiver НЕ демонстрирует пробитие фронтир-судьи.** 89-99 % — это Openchat/Mistral/Llama-2/Llama-3 7-8B + оптимизационная атака. Вывод «наш Claude-судья хакается» опирается не на неё, а на null-модель (GPT-4-аннотатор, чёрный ящик, **без оптимизации** — 76.8 %) и Preference Manipulation (продакшн Bing/Perplexity + Claude 3 Opus). Это надо держать в голове: **самое сильное свидетельство против нашего судьи — S1-ICLR-работа про бенчмарки, а не «профильная» security-работа**.
5. **«Adaptive attacks break defenses» (2503.00061) — indirectness = large.** Vicuna-7B / Llama3-8B + GCG/AutoDAN (градиенты). Против Claude по API так не атакуют. Качественный урок переносится, конкретные >50 % — **нет**.
6. **AutoDojo (2606.15057) — свежий (июнь 2026) непрорецензированный препринт.** Я взял его абстракт дословно; **ячейки таблиц не верифицировал**, названия пяти целевых моделей в абстракте не раскрыты. Его результат про «action-open» — самый решение-меняющий во всём обзоре, и одновременно **самый слабо подтверждённый**. Нужна независимая проверка (или наш собственный тест).
7. **ГЛАВНЫЙ ПРОБЕЛ: нет базовой ставки.** Я **не нашёл ни одной работы**, измеряющей, **как часто prompt injection реально встречается на обычных страницах**, которые прочитает исследовательский агент. Все ASR — условные: *«при условии, что страница адверсариальна»*. Без базовой ставки нельзя посчитать ожидаемый ущерб и, значит, нельзя доказательно выбрать между «дешёвый гейт» и «дорогая архитектура». Косвенно: экономический стимул доказан (Preference Manipulation + индустрия GEO), реальные CVE есть (EchoLeak, CurXecute CVE-2025-54135 в Cursor, CVSS 9.8) → ставка **не ноль**. Но это не измерение. **Это надо мерить самим (TEST-3).**
8. **Заявленный «0.4 % FPR / 5.7 % FNR» у классификаторов Claude Code auto-mode** встретился только во вторичном блоге; на первоисточнике Anthropic я его **не подтвердил** → в находки не вношу.
9. **Нет измерений эффективности карантина-суммаризации.** Claude Code WebFetch по факту реализует Dual-LLM (отдельное контекстное окно, малая модель отвечает на вопрос, а не отдаёт сырой HTML) — но **публичных ASR-чисел для этого механизма нет**. Насколько он реально режет атаку — неизвестно; архитектурно правильно, эмпирически не подтверждено.
10. **Spotlighting (Hines et al., 2403.14720) я прочитал только по вторичным упоминаниям** («<2 % ASR»), первичную работу не открывал. Контраст «заявлено <2 % vs измерено 41.7 % на AgentDojo» держится на измеренной стороне (она первична и надёжна); заявленную сторону цитирую как **непроверенную**.

## Короткий ответ

**Indirect prompt injection против агента, читающего веб, — это не гипотетика: 33.9 % (Claude 3.5 Sonnet) и 47.7-57.7 % (GPT-4o) targeted ASR на 629 реалистичных security-кейсах AgentDojo, +24 %→47 % от одной дописанной строки в InjecAgent, и «no browser agent is immune» от самого Anthropic (остаточные 11.2 %, ~1 % под адаптивной атакой).** Работает ровно один класс защит — **архитектурная изоляция** (tool filter: 57.7→6.8 % и полезность **выросла**; CaMeL: доказуемая безопасность ценой 7 п.п.); промпт-обёртки половинчаты (разделители 57.7→41.7 %), а детектор инъекций — **худшая покупка** (−27.5 п.п. полезности и первым падает под адаптивной атакой). **Но главное для нас: LLM-судья, грейдящий страницы, — самостоятельная мишень, которую НИ ОДНА из этих защит не прикрывает** (у судьи нет инструментов — а атака на суждение их и не требует): нулевая модель берёт **76.8 % у GPT-4-судьи вообще без оптимизации**, а на продакшн-системах (Bing, Perplexity, Claude 3 Opus) скрытый текст на странице поднимает выбор атакующего до **2.5-7.2×** и **>90 %**. Отсюда дизайн: рубрика фиксируется **до** выхода в сеть, инструменты — **до** первого байта из сети, выход судьи — **схема, а не проза**, и **ни один байт веб-происхождения не попадает в авто-загружаемые CLAUDE.md/память без гейта** — иначе модуль превращается в машину по отмыванию недоверенного текста в привилегированный инструкционный канал, навсегда.
