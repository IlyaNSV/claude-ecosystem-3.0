## 0. ГЛАВНЫЙ ВЫВОД — отрицательный, и это честный результат

**Литература НЕ поддерживает интервенцию «сократить CLAUDE.md».**

Не «поддерживает слабо». Не «поддерживает с оговорками». **Не поддерживает.**

| Аргумент, ради которого затевался аудит | Вердикт после Прогона 2 |
|---|---|
| «Длинный CLAUDE.md стоит денег каждый ход» | ☠️ **МЁРТВ.** Под кэшем ~19k резидентных токенов = **$0.0095/ход**. Резка вдвое экономит **$0.006/ход**. На подписке — **$0.00** |
| «Длинный CLAUDE.md ломает соблюдение правил» | ☠️ **АФФИРМАТИВНЫЙ НУЛЬ** на 25–500 строк (McMillan, 1 650 сессий **Claude Code CLI**, BF10 = 0.05–0.10, т.е. 10–20× свидетельства **В ПОЛЬЗУ** H0) |
| «Позиция/порядок правил критичны» | ☠️ **НУЛЬ** (McMillan) + primacy ≈ 0 на нашей плотности (IFScale, 20 моделей вкл. Claude Opus 4) |
| «Противоречия global↔project дороги» | ☠️ **АФФИРМАТИВНЫЙ НУЛЬ** (McMillan, BF10 0.05–0.10) |
| «Меньше контекста → лучше поведение» | ☠️ **ОПРОВЕРГНУТ** там, где надо перебить дефолт модели (Many-Shot ICL, NeurIPS'24 spotlight) |
| «Перестройка контекста улучшит КАЧЕСТВО работы» | ☠️ **ОПРОВЕРГНУТ 3 независимыми работами.** Интервенции в среду агента двигают **стоимость и навигацию**, не **способность** |

**Что осталось живого — и это НЕ про длину файла:**

1. **Единственный доказанный критерий реза — ИЗБЫТОЧНОСТЬ, а не объём** (Gloaguen, прямой эксперимент
   с удалением документации из репо). Ожидаемый выигрыш — **малый** (наш файл уже в основном
   указательный, pointer-collapse был сделан правильно). Резать надо блоки, дублирующие репо:
   «Что строим», дерево «Repository structure», любые снапшоты статуса.
2. **Единственный воспроизведённый живой эффект во всём корпусе — внутрисессионный распад комплаенса**
   (OR ≈ 0.944 на шаг). Лечится **не** укорачиванием файла.
3. **Единственная интервенция с измеренным подъёмом в 20 пунктов — детерминированный гейт**
   (проза 67.0% → guardrails 88.3%). **Наши 🔒-гейты — математически необходимый механизм, а не
   бюрократия.**
4. **Самая дешёвая и самая точная находка прогона получена бесплатно:** статический линт рулбейса
   (Preece) нашёл **3 реальные структурные аномалии в ITP** — 2 амбивалентности + 1 непокрытая ячейка
   гейта, причём самая опасная (необратимое действие при неполной уверенности → **не срабатывает ни
   одно правило**).

**Самое неприятное для нас:** мы **не сможем измерить себя из этой ситуации** на соло-девском объёме.
MDE при 600 прогонах агента ≈ **15 п.п.** Литературные работы использовали 1 650 сессий / 16 050
наблюдений (McMillan) и 60 000 траекторий (шум агентных эвалов). Наш реальный измерительный бюджет
хватает ровно на две вещи: **per-rule compliance через детерминированные чекеры** (дёшево, потому что
единица — правило, а не задача) и **статический линт**. На «успех задач» — не хватает и не хватит.

## 2. НЕМЕДЛЕННЫЕ НАХОДКИ (пилот линта УЖЕ прогнан, это не гипотезы — это дефекты в тексте)

**A1-1. Амбивалентность T0 × T2** (глобальный CLAUDE.md, ITP шаг 2)

**D1-1. Непокрытая ячейка гейта ITP — и она САМАЯ ОПАСНАЯ**
