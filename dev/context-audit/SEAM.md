## 1. Что литература поддерживает ВМЕСТО реза

**Инженерию принуждения, а не редактуру файла.**

1. **Арифметика конъюнкции.** ManyIFEval (Findings EMNLP 2025): у Claude соблюдение *каждого* правила держится **0,95 → 0,93** при n=1→10, а соблюдение *всех сразу* падает **0,95 → 0,48**. Модель чистой независимости предсказывает это с **MAE 0,02, r=0,994** ⇒ «обвал» — **произведение вероятностей**, не поломка модели. Проза даёт 60–68% комплаенса (ContextCov: 67,0%); детерминированный гейт — 100%.
   ⇒ **Единица учёта — число ALWAYS-ON правил, а НЕ длина файла.** Условно-триггерные правила почти бесплатны.
   ⇒ **🔒-гейты — механизм, а не бюрократия. Расширять.**
2. **Явный триггер + scope В ТЕКСТЕ у каждого правила.** Вендорский гайд про Opus 4.8 дословно: *«does not silently generalize an instruction from one item to another… state the scope explicitly»*. Control Illusion (AAAI-2026): placement (system vs user) почти не важен против **маркировки ограничения**; Claude-3.5-Sonnet следует назначенно-приоритетной инструкции лишь **20,3%**. Preece 1994: *«knowledge … "hidden" in the ordering of the rules»*.
   ⇒ **Полагаться на порядок в файле, на тон (IMPORTANT / MUST / 🔒) или на «оно же специфичнее» — ЗАПРЕЩЕНО.**
3. **Ни байта веб-происхождения в авто-загружаемый канал без гейта.** У LLM-судьи нет инструментов ⇒ ни одна работающая защита его не прикрывает, а его суждение и есть consequential action. Запись в CLAUDE.md/память **персистентна**. (AgentDojo — NeurIPS 2024 D&B; InjecAgent — Findings ACL 2024; Greshake — AISec@CCS 2023.)
4. **Детерминированные чекеры вместо LLM-судьи там, где правило машинно-проверяемо.**

### (историческая справка) Что было не так

**Дефекты** (проверены по буквальному тексту `~/.claude/CLAUDE.md`, подтверждены дважды):

- **D1-1 (опасный).** При `{p ∈ [0.6, 0.85), B ≤ 2, R = дорого/необратимо}` **не срабатывает НИ ОДНО правило**: T0 требует `p ≥ 0.85` (второй дизъюнкт требует R=дёшево); T1 требует R=дёшево; T2 требует `p < 0.6` ИЛИ (`B ≥ 3` И R=дорого). ⇒ **Необратимое действие при неполной уверенности — поведение НЕ ОПРЕДЕЛЕНО.**
- **A1-2.** При `{p < 0.6, R = дёшево-обратимо}`: Шаг 2 велит **спросить** (T2), Шаг 5 **запрещает**. Precedence подразумевается **расположением** — ровно дефект Preece.

**Предлагаемая замена Шага 2** (гейт становится **тотальным**, precedence объявлена **текстом**):

```markdown
**Шаг 2 — гейт. Сначала R, потом p. Правила упорядочены; первое подошедшее выигрывает.
Precedence объявлена ЗДЕСЬ ЯВНО, а не подразумевается расположением текста.**

| R | условие | уровень |
|---|---|---|
| **дорого / необратимо** | `p ≥ 0.85` **И** `B ≤ 2` | **T0** — делать молча |
| **дорого / необратимо** | **иначе** (`p < 0.85` ИЛИ `B ≥ 3`) | **T2** — сначала спросить |
| **дёшево-обратимо** | `p ≥ 0.85` | **T0** — делать молча |
| **дёшево-обратимо** | `p < 0.85` | **T1** — действовать + зафиксировать допущение одной строкой |

**Инвариант (проверяемый):** таблица ТОТАЛЬНА — покрывает всё пространство (R × p × B).
- Необратимое действие при неполной уверенности **НИКОГДА** не идёт молча.
- На дёшево-обратимом **НИКОГДА** не спрашиваем (цена вопроса выше цены отката).
```

**И замена Шага 5** (перестаёт нести precedence — становится входом в гейт, а не исключением из него):

```markdown
**Шаг 5 — ПЕРЕД применением гейта подними `p` дешёвыми средствами.** Это НЕ исключения из гейта,
а вход в него:
- ответ выводим из контекста / кода / истории → выведи, `p` вырастет;
- есть конвенциональный дефолт → возьми его (упомяни одной строкой), `p` вырастет;
- пользователь уже отвечал это в сессии или в памяти → `p` вырастет.

Если после этого `p < 0.85` **И** R = дорого/необратимо — гейт даёт **T2**. Это НЕ обходится.
```

**Проверка тотальности** (почему патч закрывает дыру):

| ячейка | старый ITP | новый ITP |
|---|---|---|
| `p=0.95, B=1, R=необратимо` | T0 | **T0** (без изменений — шума не добавляем) |
| `p=0.70, B=2, R=необратимо` | **НИЧЕГО** ⛔ | **T2** ✅ *(D1-1 закрыт)* |
| `p=0.50, B=2, R=дёшево` | T2 ⟷ Шаг 5 запрещает ⛔ | **T1** ✅ *(A1-2 закрыт)* |
| `p=0.90, B=1, R=дёшево` | T0 | **T0** |
