# DEFERRED — Векторный RAG-слой над продуктовым корпусом

> **Статус:** DEFERRED by design (DEC-DEV-0226, 2026-07-23). Решение принято при
> проектировании `/product:browse` + `/product:ask` (`dev/product-browse/DESIGN.md`).

## Что отложено

Векторный/embeddings-слой семантического поиска по `.product/` (и шире — по нескольким
проектам): индексация артефактов, retrieval по векторной близости для `/product:ask`.

## Почему отложено

Корпус `.product/` мал (десятки-сотни файлов) и жёстко структурен: канонические ID
(FM-001, SC-002…), 24 типа, frontmatter со status/confidence. На таком корпусе
**структурный retrieval** («лестница добора» `skills/product/corpus-qa.md`: маршрутизация →
тип-роутер → frontmatter-фильтр → grep → чтение) точнее и дешевле векторного: нет
инфраструктуры (векторное хранилище, синхронизация индекса), нет embeddings-затрат,
нет расхождения индекса с git-состоянием. Retrieval помогает на редких фактах в больших
корпусах — не на малых структурированных.

## Bring-forward триггеры (любой из)

1. **≥2 подтверждённых промаха лестницы добора** на реальных вопросах владельца:
   ответ существовал в корпусе, но структурный путь (тип-роутер + grep) его не нашёл.
2. **Мультипроектный корпус**: потребность спрашивать поверх нескольких `.product/`
   одновременно (портфельные вопросы), где ручная маршрутизация по проектам не масштабируется.
3. Корпус одного проекта вырос настолько, что notes/lessons/DA-findings перестали
   находиться grep-ом (сигнал: `/product:ask` регулярно уходит в `--deep` fan-out).

## Заготовка направления (зафиксировано веб-добором 2026-07-23, Informed Fetch)

- Claude Code сам не индексирует корпус — путь через **MCP-сервер векторного поиска**:
  кандидаты на момент фиксации — chroma-mcp (официальный Chroma), mcp-server-qdrant
  (официальный Qdrant), claude-context (zilliztech, hybrid BM25+dense), obsidian-notes-rag
  (community, именно markdown-vault). Актуальность кандидатов при bring-forward — перепроверить
  (informed-fetch), ландшафт MCP меняется быстро.
- Embeddings: у Anthropic собственного API нет — официальный партнёр Voyage AI;
  локальная альтернатива — Ollama-embeddings (без внешнего API).
- Второй чат-UI (AnythingLLM / Open WebUI) НЕ заводить — Q&A-интерфейс остаётся
  Claude Code (`/product:ask`), RAG-слой лишь расширяет его retrieval.

## Что НЕ является триггером

Желание «чтобы был RAG» без наблюдаемого промаха структурного добора — не триггер
(симметрично phantom-audience guard DEC-DEV-0046 у Phase D wiki).
