# CNT-001 — product-module → cc-sdd

**Last verified:** 2026-05-26 (with --allow-draft against FM-001 fixture)

## Purpose

PMO zone **D2-B02 (Feature Specification)** is owned by the Product Module — it produces
`FM-NNN-handoff.md` snapshots embedding the full behavioral spec (FM, SC, BR, LC, VC,
IC, NFR, RPM). PMO zones **D2-T01 (Architecture), D2-T04 (API Contracts),
D2-T06 (Task Decomposition)** are owned by cc-sdd via its `/kiro-spec-*` skills.

This contract bridges the handoff boundary: it transforms a Product Module handoff into a
`.kiro/specs/<feature>/brief.md` that cc-sdd's `/kiro-spec-init` reads to skip
interactive clarification and proceed directly to architecture and task generation.

## When this contract is used

1. Product Module produces handoff: `/product:handoff FM-NNN` → `.product/handoffs/FM-NNN-handoff.md`
2. Run adapter:
   ```
   node .claude/integrator/adapters/handoff-to-ccsdd.js --fixture .product/handoffs/FM-NNN-handoff.md
   ```
   (add `--allow-draft` if handoff status is partial/draft)
3. Adapter writes `.kiro/specs/<slug>/brief.md` and prints `next_command: "/kiro-spec-init <slug>"`
4. Run `/kiro-spec-init <slug>` in Claude Code — cc-sdd picks up `brief.md` and skips clarification

## Producer side — Product Module handoff

| Field | Source | Used for |
|---|---|---|
| `frontmatter.id` | HANDOFF-FM-NNN | Contract guard C-01 |
| `frontmatter.type` | "feature-handoff" | Contract guard C-02 |
| `frontmatter.feature` | FM-NNN | Slug generation + C-03 |
| `frontmatter.title` | Feature title | Slug + brief.md header |
| `frontmatter.status` | ready/partial/draft | Draft guard C-06 |
| `frontmatter.dor_validation_passed` | bool | Warning C-07 |
| `frontmatter.blocking_issues` | array | Guard C-08 (must be empty) |
| `body §Executive Summary` | markdown section | brief.md Problem section |
| `embedded_artifacts.scenarios/business_rules/nfr` | arrays | brief.md Boundary Candidates list |
| `artifact_hashes` | sha256 per artifact | Reference for drift detection (not transformed) |

## Consumer side — cc-sdd `/kiro-spec-init`

- Skill file: `.claude/skills/kiro-spec-init/SKILL.md`
- Execution flow: reads `.kiro/specs/<feature-name>/brief.md` if exists → pre-fills project description → skips clarification questions → creates `spec.json` + `requirements.md`
- Slug format from adapter: `fm-{nnn}-{first-4-words-of-title-slugified}` (e.g., `fm-001-authentication-accounts`)

## Field mapping

| Producer field | Transform | Consumer artifact |
|---|---|---|
| `feature` (e.g., FM-001) | featureSlug() — lowercase + first 4 title words | `.kiro/specs/<slug>/` directory name |
| `title` | strip "— handoff" suffix; embed verbatim | brief.md `## Problem` heading |
| `body §Executive Summary` | regex extract H2 section | brief.md `## Problem` body |
| `embedded_artifacts.scenarios[]` | bullet list | brief.md `## Boundary Candidates` |
| `embedded_artifacts.business_rules[]` | bullet list | brief.md `## Boundary Candidates` |
| `embedded_artifacts.nfr[]` | comma-joined | brief.md `## Constraints` |
| `current_product_tier` | direct | brief.md `## Constraints` |
| `mode` | direct (info only) | brief.md metadata comments |

## Contract checks (validateContract function)

| ID | Level | Check |
|---|---|---|
| C-01 | error | id matches `^HANDOFF-FM-\d{3}$` |
| C-02 | error | type == "feature-handoff" |
| C-03 | error | feature matches `^FM-\d{3}$` |
| C-04 | error | title non-empty |
| C-05 | error | status field present |
| C-06 | error | status != draft|partial OR `--allow-draft` |
| C-07 | warn | dor_validation_passed == true |
| C-08 | error | blocking_issues is empty array |
| C-09 | error | body section non-empty (>0 chars after frontmatter) |
| C-10 | warn | embedded_artifacts object-fields not deep-parsed (advisory) |

## Failure modes — runbook

| Symptom | Cause | Action |
|---|---|---|
| Exit 1, C-06 fail | Handoff is draft/partial | Run `/product:handoff FM-NNN` without `--mode draft`; retry adapter |
| Exit 1, C-08 fail | blocking_issues non-empty | Resolve listed blockers in handoff; regenerate |
| Exit 2, parse error | Frontmatter malformed | Verify handoff has `---` delimiters and valid YAML |
| ENOENT writing brief.md | Wrong working directory | Run adapter from project root |
| SyntaxError | Node < 14 | Upgrade Node to >=14 (project on v18.20.4) |

## Drift triggers

This contract becomes stale if any of the following happen:

- Product Module changes HANDOFF-FM frontmatter schema (renames fields, adds required fields)
- HANDOFF-FM schema version bumps (1 → 2)
- cc-sdd updates kiro-spec-init's brief.md intake format
- FM naming convention changes (e.g., FM-NNNN with 4 digits instead of 3)
- cc-sdd major version bump (v3 → v4) — re-verify all skill names and brief.md handling
