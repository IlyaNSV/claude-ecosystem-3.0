### 🟡 Warning

**[G · confidence high] Autolog не срабатывает на `Edit`/`Write` integrator-файлов на Windows**
— `.claude/hooks/integrator/journal-hook.js`

Хук зарегистрирован на `PostToolUse` с matcher, покрывающим `Edit`:
```
"matcher": "Bash|Write|Edit|NotebookEdit",
"command": "node .claude/hooks/integrator/journal-hook.js"
```
а `INTEGRATOR_PATH_PATTERNS` явно включает `active-tools.yaml`:
```
/\.claude\/integrator\/active-tools\.yaml$/,
```
Значит, `Edit` `active-tools.yaml` (timeline #2, `2026-05-26T12:26:51Z`) **должен был** дать
`#auto`-запись. Эмпирически — не дал: после правки mtime журнала остался прежним
(`2026-05-26 13:50:03 +0300` = `10:50:03Z`, проверки #7 и #11), а финальный `tail` (#13)
перескакивает от ручной `DEC-INT-0001` (10:50Z) сразу к записи `npx` (12:33Z) — записи `Edit`
нет вовсе.

Дифференциал решает root cause: тот же хук **сработал** на Bash `npx` (#12→#13 добавил
`## [2026-05-26T12:33:51.849Z] #auto … Subject: npx cc-sdd@latest --help`), то есть хуки в
сессии активны. Различие — в `classifyAction`: для `Edit` путь матчится регэкспами с
**прямыми слэшами** (`\.claude\/integrator\/…`), а `file_path` в Edit пришёл с
**бэкслэшами** (`C:\…\my-first-test\.claude\integrator\active-tools.yaml`, timeline #2) на
платформе `win32`. `pat.test(p)` → `false` → `classifyAction` возвращает `null` → записи нет.
Bash-ветка не зависит от разделителя пути (матчит строку команды `^\s*npx\s+`), поэтому
работает. Косвенное подтверждение: в `project-journal.md` **все** существующие `#auto`-записи
имеют `**Tool:** Bash` — ни одной `Write/Edit`-записи за всю историю проекта.

Эффект: на Windows правки `active-tools.yaml`, `pmo-mapping.yaml`, контрактов, адаптеров через
`Edit`/`Write` **не журналируются** — теряется audit-trail для значимой части modifying-действий
Integrator (SPEC §6: журнал «на каждое modifying действие»). Empirical-факт «не залогировано» —
high; указанная причина (path-separator) — сильная инференция с прямыми доказательствами обеих
сторон, поэтому confidence high. Рекомендация (не применяю — auditor read-only): нормализовать
путь перед матчингом (`p.replace(/\\/g, '/')`) либо сделать регэкспы separator-agnostic.
