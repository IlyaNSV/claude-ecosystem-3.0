## §0 Контракт исполнения (сквозной, для каждого пункта)

1. **Inspection brief перед заимствованием** (borrow-not-adopt, прецедент bmad-loop
   DEC-META-0002): recon-субагент читает донора → извлекает паттерн/инварианты →
   main ревьюит бриф → реализация СВОЯ, чужой код не копируется. MDP: recon=sonnet,
   реализация=opus, бриф+ревью=main.
2. **Soft-миграции** (CLAUDE.md §6): absent == старое поведение 1:1; состояние пилота
   не вайпается; новые ноги enforcement входят warn-only → strict по решению владельца
   (инвариант 4 трека когерентности).
3. **DoD пункта:** тесты + verify зелёный + смоук на пилоте (если трогает consumer-zone)
   + DEV_JOURNAL при tradeoff ≥2 вариантов + CHANGELOG при consumer-zone.
4. **Cuttable-scope фильтр** перед стартом каждого пункта: «что урезать, чтобы получить
   feedback быстрее?»
5. **«Ослеп ≠ нашёл»** для всех новых чекеров: внутренняя ошибка → громкий warn + exit 0;
   аварийный тумблер env у каждого strict-гейта.
