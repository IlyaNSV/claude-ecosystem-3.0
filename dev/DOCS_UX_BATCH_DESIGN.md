# Docs UX Batch — Design (proposal / pre-decision)

> **Статус:** proposal, **до** DEC-DEV. Зеркалит паттерн [`dev/ECOSYSTEM_VISION.md`](ECOSYSTEM_VISION.md) —
> pre-decision дизайн-документ, **не трогающий ни одного consumer-файла**. На одобрении
> направления → DEC-DEV-NNNN + нарезка на волны + исполнение. Дата: 2026-07-01.
>
> **Что это:** сквозной дизайн доработки документации (интерактивной `docs/guide/*.html` +
> обычной `docs/guide/*.md`) батчем — пункты A–E из proposal-волны. Организован не как список
> фич, а вокруг **психологии человека, строящего ментальную модель системы по уровням масштаба**.

## 0. TL;DR

Три архитектурных «столпа» + отдельные контентные доки, все под одним анти-дрейф-зонтом:

- **Столп I — «Одна дверь, слоями»** (A1/A2/A3/B1): единый вход `docs/guide/README.md` → лестница
  понимания **L0→L5**, Diátaxis-метки на каждой доке, роутер «Я хочу…», оба интерактивных карты
  на видном месте. Прочие хабы (root README, docs/MAP, HOME) → однострочный указатель на эту дверь.
- **Столп II — «Общий интерактивный шелл»** (C1/C2/C3): вынести общий chrome двух карт в
  вендоренный `map-shell.{js,css}` — навбар «переключить вид», **единый hash-deeplink с `view=`**,
  **in-app UTF-8 doc-панель** (чинит кракозябры в ОБЕИХ картах + битый путь), единая клавиатура/легенда.
- **Столп III — «Генерируемый слой узнавания и провенанса»** (E1/D2/A3): `gen-glossary.cjs`
  (SSOT = H1 артефактов + overlay-глоссарий) → `03-glossary.md` + `glossary.json` (тултипы в картах);
  единый провенанс-штамп на всех генерируемых доках; Diátaxis `--check`.
- **Отдельно:** D1 «Зачем гейты» (`06-gates.md`); B2 генерируемая «анатомия» одной картинкой;
  C4 «Ты здесь» (project-state) + C5 «объясни путь» — крупные P2-фичи карт; E2 аудит словаря.

Порядок: **Волна 0** (P0/P0.5 quick wins) → **Волна 1** (ядро понимания) → **Волна 2** (крупные ставки).
Всё генерируемое получает `gen:X` + `gen:X:check` + место в `npm run verify`; фикс кодировки получает
**regression-тест** (панель декодирует UTF-8) — батч не превращается в дрейф-долг.

## 2. Принципы дизайна (психологические ограничения — явно)

Батч подчинён шести когнитивным ограничениям. Это не украшение — каждый пункт A–E обязан служить ≥1:

1. **Согласование ментальных моделей (Norman).** Дока обязана проговаривать *модель системы* явно,
   чтобы модель пользователя с ней совпала. Расхождение = путаница (частый кейс: D6 governance ↔ D7 dev-only).
2. **Бюджет когнитивной нагрузки (Sweller).** Резать *экстранейную* нагрузку (навшум, кракозябры,
   разбросанные входы); *интринзик* (24 артефакта, 44 правила, 6 доменов) дозировать через
   **progressive disclosure**, а не вываливать разом.
3. **Разделение по Diátaxis.** Tutorial / How-to / Reference / Explanation — у каждой доки одна доминирующая
   роль. Читатель должен знать *зачем* страница, не искать how-to в reference.
4. **Recognition over recall (Nielsen).** Узнавать (тултипы, глоссарий, поиск, карты), а не вспоминать
   акронимы. Разгрузка рабочей памяти.
5. **Анти-дизориентация.** Один пространственный якорь («Начни здесь»), куда всегда можно вернуться —
   против «lost in hyperspace».
6. **Снижение тревоги + доверие.** Легибельные состояния отказа (гейты как *страховка*, «что делать
   дальше») + провенанс-штамп («сгенерировано из X, проверено в CI») = доверие к актуальности.

## 3. Архитектура

### 3.1. Репо-нативные ограничения (не нарушать)

- **Трёхслойная модель.** Всё пользовательское → USER-слой `docs/guide/`; генераторы → `dev/meta-improvement/scripts/`;
  дизайн-доки → `dev/`. Не смешивать с REFERENCE (`docs/pmo/…`, спеки) и DEV (`CLAUDE.md`).
- **SSOT + анти-дрейф.** Всё, что *выводимо* из источника, — **генерировать**, не вести руками; каждый
  генератор = `gen:X` + `gen:X:check` (neutralize date/sha) + `--selftest` + место в `npm run verify`.
  Паттерн скопировать 1:1 с `gen-ecosystem-map.cjs` / `gen-process-map.cjs` (recon §2–6).
- **Оффлайн.** Ни одной сетевой зависимости; всё вендорится в `docs/guide/vendor/` (как Cytoscape/ELK).
- **Русская копия.** Весь текст — по-русски.
- **Не ломать состояние пилота.** Никаких изменений формы `.product/`; C4 (project-state) — read-only оверлей.

### 3.2. Три столпа → пункты

```
Столп I  «Одна дверь, слоями»      → A1 хаб · A2 «Я хочу…» · A3 Diátaxis · B1 лестница L0–L5
Столп II «Общий интерактивный шелл»→ C1 UTF-8 doc-панель(+фикс пути) · C2 дешум · C3 интеграция карт
Столп III«Генерируемое узнавание»  → E1 gen-glossary · D2 провенанс-штамп · A3 doc_type --check
Отдельно                           → D1 «Зачем гейты» · B2 анатомия(gen) · C4 «Ты здесь» · C5 «объясни путь» · E2 аудит словаря
```

## 4. По-пунктный дизайн (A–E)

> Формат: **намерение · психика · дизайн (файлы/контракт/механизм) · анти-дрейф/тест · усилие (S/M/L) · зависимости · открытая развилка.**

### A1 — Единый вход «Начни здесь» _(Столп I)_
- **Намерение:** одна дверь, собирающая территорию доки.
- **Психика:** #5 анти-дизориентация, #2 экстранейная.
- **Дизайн:** пересобрать `docs/guide/README.md`: (0) 60-сек «что это» + 1 картинка (ссылка на B2),
  (1) лестница L0–L5 (B1) — каждый уровень линкует нужную доку/карту, (2) блок «Я хочу…» (A2),
  (3) обе карты на видном месте (чинит P-2). Прочие хабы (root README, docs/MAP, HOME) → однострочный
  указатель «новичок? → docs/guide/README». Коллапс 5 хабов → 1 canonical + 4 pointer.
- **Анти-дрейф/тест:** лёгкий `--check` в новом `gen-guide-index` ИЛИ (проще) `check-guide-orphans`:
  каждая `guide/*.md` упомянута в README и наоборот (анти-orphan). Список файлов — генерировать из
  frontmatter (title+doc_type+summary), прозу-рамку оставить редакторской (гибрид, как overlay).
- **Усилие:** M. **Зависимости:** A3 (doc_type), B1 (ладдер-текст). **Развилка:** D-1 (см. §7).

### A2 — Роутер «Я хочу…» _(Столп I)_
- **Намерение:** задача → точная дверь (how-to слой, которого почти нет).
- **Психика:** #4 recognition, #3 Diátaxis (how-to).
- **Дизайн:** переиспользовать существующий паттерн `ecosystem-map` — там уже есть `.task[data-cmd]` +
  18 записей `{goal, cmd}` в overlay (recon §4). Для guide-доки: markdown-таблица «Я хочу … → доку/команду/карту».
  Записи — редакторские (в overlay или в самом README). В картах — тот же `.task`-механизм (уже есть в
  ecosystem-map; добавить в processes через общий шелл C3).
- **Анти-дрейф/тест:** если записи ведут на команды — selftest «каждый cmd существует» (паттерн ecosystem-map check [12]).
- **Усилие:** S. **Зависимости:** —. **Развилка:** —.

### A3 — Diátaxis-метки _(Столп I + III)_
- **Намерение:** у каждой доки видна её роль.
- **Психика:** #3 Diátaxis.
- **Дизайн:** добавить frontmatter `doc_type: tutorial|how-to|reference|explanation` (+ опц. вторичный тег)
  в каждую `guide/*.md`. Хаб-генератор читает и группирует/подписывает. Наши доки — гибриды
  (00-concepts = explanation+reference+glossary) → метить по **доминирующей** роли + вторичный тег.
- **Анти-дрейф/тест:** `--check`: каждая `guide/*.md` имеет валидный `doc_type` из enum.
- **Усилие:** S. **Зависимости:** —. **Развилка:** D-5.

### B1 — Лестница масштаба L0→L5 _(Столп I)_
- **Намерение:** понимание дозируется по уровням.
- **Психика:** #2 progressive disclosure, #1 леса ментальной модели.
- **Дизайн:** секция в хабе (A1): **L0** «что это в абзаце+картинке» → **L1** поток D1–D6 → **L2**
  модули/артефакты/гейты → **L3** «прогнать сессию» (01-first-session) → **L4** спеки/правила (pmo/) →
  **L5** «как устроена» (dev/CLAUDE). Каждый уровень = один экран + ссылка глубже. Существующее не
  дублировать: L1/L2 линкуют 00-concepts §2–3, L3 → 01-first-session и т.д.
- **Анти-дрейф/тест:** редакторский текст; покрывается anti-orphan из A1.
- **Усилие:** M (в основном письмо). **Зависимости:** A1. **Развилка:** —.

### C1 — In-app UTF-8 doc-панель + фикс битого пути _(Столп II, **P0**)_
- **Намерение:** клик по doc-ссылке открывает содержимое читаемо, без кракозябр; чинит битый путь.
- **Психика:** #2 экстранейная (кракозябры — чистый шум), #6 доверие.
- **Дизайн:** перехватывать клики по doc-ссылкам (footer + side-panel `d.doc` + в ecosystem-map
  `c.source`/SPEC-ссылки). Вместо навигации в сырой `.md`: `fetch(relPath)` → `new TextDecoder('utf-8').decode(buf)`
  → лёгкий markdown-рендер в боковую панель. Хост-страница UTF-8 + мы декодируем сами → кракозябры
  **невозможны** (проверено: fetch+TextDecoder дал корректный текст). Путь резолвить корректно
  (`d.doc` → префикс `../../`, чинит P-5). **Fallback:** если `fetch` упал (file:// без флага) —
  открыть ссылку как сейчас (грациозная деградация; на IDE-сервере `http://localhost:63342` fetch
  same-origin работает — это твоя среда).
- **Анти-дрейф/тест:** **regression в smoke** — открыть панель на `pmo/processes.md`, ассертить, что
  в тексте есть «Версия/Назначение» (кириллица не побита) и что путь резолвится (нет 404). Ставит
  «кракозябры не вернутся» на автопилот.
- **Усилие:** S–M. **Зависимости:** — (в P0 живёт в шаблоне процессов; в C3 переезжает в общий шелл).
  **Развилка:** D-2 (fidelity рендера).

### C2 — Дешум остаточного шума _(Столп II)_
- **Намерение:** первый кадр читается за 2 сек.
- **Психика:** #2 экстранейная, закон Хика.
- **Дизайн (переформулировано по факту кода):** overlay-слои produces/… **уже** off (P-7) — не трогаем.
  Реальный остаток: (a) длинные кросс-лейн `delegate`-рёбра (оранжевый пунктир) через весь холст —
  притушить до hover/выбора (fade-unless-connected), либо бандлить; (b) легенда налезает на подпись
  «Артефакты · data objects» — сдвинуть/сжать. Плюс опция «фокус»: при выборе узла гасить несвязанные
  рёбра (переиспользовать `has-hl`-паттерн ecosystem-map, recon §5).
- **Анти-дрейф/тест:** smoke-ассерт «легенда не перекрывает подпись дорожки» (bbox-overlap как в
  текущем lane-overlap чек-е).
- **Усилие:** S. **Зависимости:** —. **Развилка:** —.

### C3 — Интеграция двух карт: общий шелл _(Столп II)_
- **Намерение:** одна когнитивная среда вместо двух приложений.
- **Психика:** #5 анти-дизориентация, #1 единая модель.
- **Дизайн:** вынести общий chrome в вендоренные `docs/guide/vendor/map-shell.js` + `map-shell.css`,
  которые **инклюдят оба шаблона**: (1) навбар «⇄ переключить вид: команды ⇆ процессы»; (2) **единый
  hash-deeplink** — расширить существующую схему ecosystem-map (`#q=&stage=&module=&status=&hl=`,
  `writeHash`/`readHash`/`suppressHash`, recon §1) полем **`view=map|processes`** + общим `focus=kind:id`,
  чтобы ссылка из одной карты открывала конкретный узел в другой; (3) общая клавиатура (`/` `Esc` guarded)
  + легенда + in-app doc-панель (C1). Данные для кросс-навигации уже есть: артефакты общие в обеих картах.
- **Анти-дрейф/тест:** smoke — из processes открыть `#view=map&focus=art:FM` и ассертить, что карта
  команд сфокусировала FM; и обратно.
- **Усилие:** M–L (аккуратно: трогает оба генерируемых шаблона — только шаблоны, HTML не править).
  **Зависимости:** C1 (панель переезжает в шелл). **Развилка:** D-3.

  Формат «сигнал → что значит → что делаешь» (переиспользовать таблицы 05-implementation §6).
- **Анти-дрейф/тест:** редакторский; линкуется из хаба (anti-orphan A1).
- **Усилие:** M. **Зависимости:** —. **Развилка:** —.

### D2 — Провенанс-штамп на всех генерируемых доках _(Столп III)_
- **Намерение:** «сгенерировано из X · проверено в CI · <sha>» = доверие к свежести.

- **Дизайн (переформулировано по факту кода):** карты **уже** штампуют `_build{date,sha,counts}` →

- **Анти-дрейф/тест:** уже есть neutralize date/sha в `--check` (recon §3) — распространить на новые доки.
- **Усилие:** S. **Зависимости:** —. **Развилка:** —.

### E1 — Генерируемый глоссарий _(Столп III)_
- **Намерение:** 24 акронима + сквозные термины из единого источника, линкуемы отовсюду + тултипы в картах.
- **Психика:** #4 recognition.
- **Дизайн:** новый `dev/meta-improvement/scripts/gen-glossary.cjs` по паттерну recon §1–6. **SSOT:**

- **Анти-дрейф/тест:** `gen:glossary` + `gen:glossary:check` (neutralize date/sha) в `verify`; selftest —
  каждый artifact-id имеет имя; каждый термин уникален; каждый glossary-term определён.
- **Усилие:** M. **Зависимости:** — (тултипы в картах — через C3-шелл). **Развилка:** D-4.

### B2 — «Анатомия» одной картинкой (генерируемая) _(отдельно, P2)_
- **Намерение:** одна схема = ментальная модель (USER/REFERENCE/DEV × D1–D6 × модули).
- **Психика:** #1 леса модели.
- **Дизайн:** C4-подобная схема. Кандидат — **генерировать** (Mermaid-блок в markdown, git-diffable,
