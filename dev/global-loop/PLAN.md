## 0. Цель одним контуром

Единый **global loop** end-to-end: владелец даёт интенцию → **Кондуктор** (пульт)
диспетчеризует релизный цикл нужного проекта на VM → экосистема итерирует по канонам
(P3→P6 → deploy-to-stage) → **DoD-предикат** меряет «релиз выкачен и готов к
использованию» → незакрытые пункты становятся очередью следующей итерации → пульт
надзирает, отвечает в мандате, батчем эскалирует владельцу → зелёный DoD + owner ratify
= выход цикла (`RL.status: released`). Два трека — две половины одного контура:
**DoD — критерий остановки внутреннего цикла проекта; Кондуктор — внешний цикл над
десятком таких проектов.**

## 2. Сшитая фазировка — волны с зависимостями

```
Волна 1 (фундамент, параллельно):   R0 ──┐            H0 ──┐
Волна 2 (автоматизация):           R1 ──┼─▶ H1 ◀──────────┘   ← сшивка №1: sweep читает DoD-отчёт
Волна 3 (событийный контур):  events.jsonl ─▶ H2 ∥ R2         ← сшивка №2: DoD-парковки = вопросы пульта
Волна 4 (масштаб):                 H3 ─▶ H4
Вне волн (event-gated):            R3 (реальный prod-хост)
```

| Волна | Единицы | Пререквизиты | Определения |
|---|---|---|---|
| **1 — Фундамент** | **R0**: термин DoD в канон + DoD-чеклист в body RL (soft) + ручная проверка на пилоте. **H0**: спайк-наблюдатель — пре-регистрация брифа (что меряем, $/sweep, stop-критерии; образец — `meta-system/SPIKE_M1_BRIEF.md`) → read-only sweep пилота → отчёт | нет | R0-R3 — `dev/release-dod/TRACK.md` §6; H0-H4 — `factory-conductor/CONDUCTOR.md` §roadmap |
| **2 — Автоматизация** | **R1**: `/product:release-dod` (read-first агрегатор + отчёт ✅/🟠/❌) → **H1**: диспетчер (intent → spawn/send на 1 проекте; вендоринг thin-waist схем меты). Сшивка №1: sweep H1+ включает DoD-отчёт | Волна 1 | |
| **3 — Событийный контур** | **events.jsonl** (хуки полосы → durable timeline; детализация — README factory-conductor §роадмап) → **H2** (Q&A в мандате) ∥ **R2** (loop-wiring: незакрытые DoD-пункты → PA/owner-queue; fabric-ячейка `release-dod`). Сшивка №2: DoD-парковки — класс вопросов, которые пульт batch-эскалирует | Волна 2; H2 жёстко gated на events.jsonl (Q&A по pane-peek запрещён) | |
| **4 — Масштаб** | **H3** (флот: реестр, WIP/внимание-бюджет, единая очередь) → **H4** (deviation watch + strengths/weaknesses ledger → feedback-outbox) | H1+H2 на 2 проектах | |

**Гейт-дисциплина:** перед каждой волной — `readiness-gate.md`; live-прогоны на VM — по
`checklists/live-run-validation.md` (executor/reviewer separation); после постройки, до
веры в интеграцию — `smoke-test-plan.md`. Смоук-долг >2 непрогнанных планов = стоп новым
built (инвариант меты №1).
