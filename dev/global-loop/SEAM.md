### Инварианты И ЗАПРЕТЫ (решены — НЕ пересматривать без новых данных)

1. **Merge в `main` ЛЮБОГО репо — только владелец.** Мандаты на merge в «Состоянии» ниже —
   сессионные, на новую сессию НЕ переносятся (при противоречии прав этот пункт).
2. **Floor autonomy непробиваем** (`prod_deploy, destructive, spend_money,
   provision_real_secret` → всегда человек); **`evt:owner.*` эмитит только владелец**
   (ANOM-OD7-2) — пульт никогда не проецирует решения владельца.
3. **Субстрат READ-ONLY** для пульта/диспетчера (уроки 54-56); **ghost-защита ввода** в
   чужие TUI-сессии: `C-u` → файл/base64 → capture-верификация → Enter (8+ ghost-случаев
   за кампанию, были опасные директивы).
4. **Анти-дублирование SSOT:** архитектура/инварианты/roadmap пульта — ТОЛЬКО
   `factory-conductor/CONDUCTOR.md`; решения — журнал DEC-DEV-0216; хартии — TRACK.md
   треков; порядок работ — PLAN.md. Вторых копий не заводить.
5. **DoD — ссылочный агрегатор** существующих вердиктов (P6/P7/Epic E/graduation/
   impl-sync/DA-RL), НЕ новый судья и НЕ новый артефакт-тип; терминатор цикла = зелёный
   DoD + owner ratify.
6. **H2 gated на events.jsonl** — Q&A-автопилот по pane-peek запрещён (канон «hook-relay,
   не pane-peek»). **R3 gated на реальный prod-хост** — не начинать.
7. **Пре-регистрация перед live-прогонами** ($/sweep и stop-критерии для H0 — до прогона,
   не после); не жечь лимиты (negative exemplar: сессия 100+ агентов).
8. **Знание экосистемы — резолвером, не пересказом** (K0, `CONDUCTOR.md` §Knowledge plane):
   лестница добора INFORMATION-MAP → генерируемые каталоги → SSOT → живое состояние;
   **version-skew** (про проект — из его `.claude/`, про канон — из host-клона);
   **ответ пульта без указателя на SSOT-источник = не ответ**. Зеркала/копии доков не заводить.
9. **Вход в стройку Кондуктора (H-фазы) — только после ассист-периода Волны 0 и явного
   «го» владельца.** До того пульт = владелец в терминале + ассистент; опыт пишется в
   `ASSIST_LOG.md` — не начинать H0 «потому что план его называет».
